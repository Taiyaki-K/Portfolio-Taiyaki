## 🎮 作品概要
「ロシアンルーレット」の緊張感に、RPG的な「スキル」と「ステータス成長」の要素を掛け合わせた、ターン性コマンドバトルゲームです。
サークルの新入生チーム制作企画として開発しました。既存のゲーム（Buckshot Roulette）をベースにしつつ、独自の駆け引き要素を加え、オリジナルのゲーム性に昇華させました。

## 👨‍💻 担当箇所と制作体制
当初はチーム制作としてスタートしましたが、進行上の課題により、最終的に私がすべてのコードの実装を行いました。
タイトルロゴやスキルアイコン等のアートワークは、2Dデザイナー1名に制作していただきました。

## 🚀 アピールポイント
* 企画から実装まで完遂: 初めて自分でゲーム内容を考案し、形にした作品です。多くの新入生チームが未完成で終わる中、期限内に「遊べるゲーム」として公開まで漕ぎ着けました。
* 独自のゲームデザイン: 「運」だけでなく、スキルやヘイト管理による「戦略」が勝敗に絡むようバランス調整を行いました。

## 💭 振り返りと反省
本作品は、私がプログラミングを始めて間もない頃に制作したため、技術的な課題が多く残っています。しかし、この経験が今後のクラス設計、オブジェクト指向、デザインパターン等の勉強への強いモチベーションに繋がっています。

### 1. 完遂を最優先した開発
当時の私は、オブジェクト指向やC#の基礎知識が不足していました。また、開発終盤は納期に間に合わせることを最優先したため、AI支援ツールのコードを十分な理解なしに使用したり、可読性の低いコード（変数の命名やコピペの多用）を量産してしまいました。
結果として「技術的負債」の重さを痛感しましたが、「どんな手段を使ってでも、動くものを納期までに出す」というエンジニアとしての粘り強さは発揮できたと考えています。

### 2. 設計（アーキテクチャ）の欠如
機能ごとのクラス分離や責務の分割（デザインパターン）を知らなかったため、コードの結合度が極めて高く、バグ修正や機能追加が困難な状態になっていると感じます。
